Cherry Picking with Subversion

2015-03-27 by fredrik svn version control

So I am back in subversion after two years with git only development. I feel insecure…. Merge is not working so well as in git. Today I needed to merge changes from a single commit on a release branch to our main branch. Cherry picking….. Turned out it was possible with subversion:

svn merge -c <revision> <source_branch>

The -c merges only the commit and nothing else. Useful!
